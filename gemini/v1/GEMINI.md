# GEMINI 任务助手上下文

本文档描述了当前目录下的项目结构、技术栈、使用方法和开发约定，旨在为未来的开发工作提供上下文。

## 目录概览

此目录 (`v1`) 包含一个独立的、单文件的待办事项 (Todo) 应用程序实现。该应用完全使用原生 Web 技术构建，无需任何构建工具或依赖。它具有独特的、大胆的视觉风格（标有 "TASK MANAGER v2.0"），包含流畅的动画效果和自定义的 UI 控件。

### 核心功能

*   **添加任务**：输入文本并添加到待办列表。
*   **切换状态**：标记任务为已完成或未完成，并配有自定义复选框交互动画。
*   **删除任务**：删除单个特定的待办事项。
*   **状态过滤**：支持按照“全部” (ALL)、“活跃” (ACTIVE)、“已完成” (DONE) 过滤显示任务。
*   **清除已完成**：一键清理所有已完成的任务 (NUKE DONE)。
*   **数据持久化**：使用浏览器的 `localStorage`（键名为 `my-todos-v2`）存储数据，确保刷新页面数据不丢失。它还包含从旧版本 (`my-todos-v1`) 迁移数据的简单逻辑。

### 关键文件

*   **`index.html`**：这是该目录下的唯一源文件，包含了完整的应用程序：
    *   **HTML 结构**：定义了应用的语义化骨架，包括标题、输入表单、控制面板（过滤器和计数器）以及待办列表容器。
    *   **CSS 样式**：内联在 `<style>` 标签中。使用 CSS 变量进行主题颜色管理，从 Google Fonts 引入了自定义字体（Bricolage Grotesque, DM Mono），并实现了复杂的视觉效果，如噪点背景遮罩、自定义复选框、以及各种状态下的过渡和动画。
    *   **JavaScript 逻辑**：内联在 `<script>` 标签中。实现了核心的 CRUD（创建、读取、更新、删除）功能、过滤逻辑以及与 `localStorage` 的读写同步。

## 使用方法

这是一个纯静态的前端应用，没有任何构建步骤。

*   **运行**：直接在任何现代网页浏览器中打开 `index.html` 文件即可运行和测试。可以通过文件系统双击打开，或者直接拖拽到浏览器窗口中。

## 开发约定

*   **技术栈**：纯 HTML5, CSS3, Vanilla JavaScript (ES6+)。
*   **数据结构**：任务对象包含三个属性：
    *   `id`：字符串，使用 `Date.now().toString()` 生成。
    *   `text`：字符串，用户输入的任务内容。
    *   `completed`：布尔值，表示任务是否已完成。
*   **UI/UX 风格**：采用粗犷的、高对比度的“新粗野主义” (Neobrutalism) 设计风格。主要使用亮红色、电蓝色和黄色，结合粗黑色的边框和不均匀的阴影效果。
*   **状态管理与渲染**：使用简单的全局变量（如 `todos` 和 `currentFilter`）进行状态维护。每次发生改变时（添加、切换、删除等），都会调用 `saveTodos()` 将状态同步到本地，并调用 `renderTodos()` 重新渲染整个列表。
*   **安全性**：实现了一个简单的 `escapeHtml` 函数，在渲染任务文本前对其进行转义，以防止跨站脚本攻击 (XSS)。
